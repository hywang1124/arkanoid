# Arkanoid C++ 期末项目最终说明

## 项目主题

项目名称：Arkanoid / Breakout 打砖块游戏

本项目使用 C++17 和 raylib 实现经典打砖块游戏。玩家控制屏幕底部的挡板，让小球反弹并击碎砖块。游戏包含分数、生命值、关卡推进、暂停、胜利和失败状态，并加入了多个用于 Demo 展示的扩展功能。

## 协作与语言规则

- `AGENTS.md` 和项目说明文档使用中文编写。
- C++ 源代码中的注释使用日语编写。
- 类名、函数名、变量名使用清晰的英文命名。
- README 使用中文为主，保留必要的英文术语，例如 `raylib`、`vector`、`GameState`。
- PPT 使用日文，标题页需要包含项目内容、成员姓名和学生 ID。


## 最终技术选择

- 语言标准：C++17
- 图形库：raylib
- 构建方式：PowerShell 脚本 `build.ps1`
- 主要运行环境：Windows + MSYS2 UCRT64 g++
- 显示方式：raylib 图形窗口
- 输入方式：实时键盘输入

选择 raylib 的原因：

- 可以直接创建窗口并绘制游戏对象，适合发表时演示。
- 键盘输入、图形绘制和帧率控制比较简单。
- 比纯控制台版本更直观，能更清楚地展示碰撞、反弹和游戏状态。

## 最终项目目标

- 完成一个可以实际运行和演示的 C++ 打砖块游戏。
- 通过类和对象展示面向对象编程思想。
- 使用 `std::vector` 管理多个砖块和多个小球。
- 实现碰撞检测、计分、生命值、游戏状态和关卡流程。
- 提供稳定的 Demo 流程和 PowerPoint 发表资料。

## 最终实现功能

### 基本功能

- 使用 `A` / `D` 或左右方向键控制挡板移动。
- 小球根据速度自动移动。
- 小球与左右墙、上墙、挡板、砖块碰撞后反弹。
- 多个砖块按网格排列。
- 砖块被击破后增加分数。
- 玩家拥有 3 条初始生命。
- 漏接所有小球后减少生命。
- 生命为 0 时进入游戏失败状态。
- 清除最终关卡的所有砖块后进入胜利状态。
- 支持重新开始和再次游玩。
- 支持暂停和继续。
- 使用 raylib 图形窗口显示游戏画面。

### 扩展功能

- 3 个关卡。
- 后续关卡会改变砖块排列。
- 后续关卡会提高小球速度。
- 砖块拥有不同耐久度，需要多次击中才能破坏。
- 支持多球。
- 支持隐藏作弊码 `thereisnospoon`。
- 作弊系统解锁后显示短暂文字提示。
- 作弊热键包括加速、生成额外小球、加宽挡板、增加生命、临时护盾和 Spoon Wave。
- 控制台输出按键日志，便于调试和 Demo 说明。

## 最终游戏状态

项目最终使用以下状态：

- `Start`：开始画面或关卡开始前等待。
- `Playing`：游戏进行中。
- `Paused`：暂停。
- `Win`：全部关卡通关。
- `GameOver`：生命值归零。

状态由 `GameState` 枚举管理。`Game` 类根据状态决定输入处理、对象更新和画面提示。

## 最终类设计

### Game 类

职责：

- 管理 raylib 窗口和主游戏循环。
- 管理当前游戏状态。
- 处理键盘输入。
- 更新挡板、小球、关卡和计时效果。
- 处理墙壁、挡板、砖块碰撞。
- 管理分数、生命、关卡推进和胜负判定。
- 绘制游戏画面、HUD、开始/暂停/胜负提示和作弊面板。

主要成员：

- `state`
- `level`
- `paddle`
- `balls`
- `bricks`
- `score`
- `lives`
- `cheatsUnlocked`
- `cheatMessageTimer`
- `widePaddleTimer`
- `shieldTimer`

### Paddle 类

职责：

- 保存挡板的位置、大小和移动速度。
- 根据键盘输入左右移动。
- 限制挡板不离开屏幕边界。
- 提供 `Rectangle` 碰撞范围。
- 支持临时改变挡板宽度。

### Ball 类

职责：

- 保存小球的位置、速度、半径和基础速度。
- 每帧根据 `deltaTime` 更新位置。
- 提供 X/Y 方向反弹。
- 支持速度标准化，保证速度大小稳定。
- 支持作弊功能中的速度倍率调整。

### Brick 类

职责：

- 保存砖块位置、大小、颜色、分数和耐久度。
- 判断砖块是否仍然存在。
- 被小球击中后减少耐久度。
- 耐久度为 0 时进入破坏状态。
- 绘制不同颜色和耐久度数字。

### Level 类

职责：

- 管理当前关卡编号。
- 创建当前关卡的砖块布局。
- 根据关卡改变砖块排列。
- 根据关卡设置砖块耐久度。
- 根据关卡提高小球速度。
- 判断是否已经是最终关卡。

## 技术难点与最终解决方法

### 小球反弹方向

难点：

- 小球撞到砖块角落时，反弹方向可能不自然。

解决方法：

- 计算小球与砖块四个方向的重叠量。
- 选择最小重叠方向决定 X 反弹或 Y 反弹。

### 挡板反弹手感

难点：

- 如果只简单反转 Y 方向，玩家很难控制小球方向。

解决方法：

- 根据小球击中挡板的位置计算偏移比例。
- 击中挡板左侧时向左反弹，击中右侧时向右反弹。
- 使用 `normalizeVelocity()` 保持速度大小稳定。

### 关卡难度

难点：

- 球速太慢会无聊，太快会难以演示。

解决方法：

- 关卡越高，小球速度越高。
- 砖块排列和耐久度逐关变化。
- Demo 时优先展示第 1 关，保证稳定。

### 状态管理

难点：

- 开始、游戏中、暂停、胜利和失败状态的逻辑容易混在一起。

解决方法：

- 使用 `GameState` 枚举明确区分状态。
- 在 `Game::update()` 中只在 `Playing` 状态更新游戏对象。
- 在 `Game::draw()` 中按状态显示不同提示。

## Demo 流程

建议 Demo 按以下顺序进行：

1. 启动程序，展示开始画面。
2. 按 `Space` 开始游戏。
3. 使用 `A` / `D` 或左右键移动挡板。
4. 击碎几个砖块，展示分数增加。
5. 故意漏接一次，展示生命值减少。
6. 按 `P` 展示暂停和继续。
7. 如时间允许，输入 `thereisnospoon` 展示隐藏功能。
8. 简短说明 3 个关卡、耐久度砖块和多球功能。
